# Overview

## Summary
### English
This wiki is a persistent second brain that compiles raw sources into linked,
maintained markdown pages. It is intended to accumulate knowledge, not just
retrieve snippets.

### 中文
这个 wiki 是一个持续维护的第二大脑，会把原始资料编译成相互链接、持续维护的
Markdown 页面。它的目标是积累知识，而不只是临时检索片段。

## Current State
### English
- The repository scaffold is in place.
- The Karpathy LLM Wiki pattern has been captured as the first seed source.
- The wiki now has module-style entrypoints for `AI News`, `AI Technologies`,
  and `AI Equities`.
- A Xiaoyuzhou podcast transcript on OpenClaw, vibe coding, and AI-native work
  has been ingested into source, concept, and analysis pages.
- Two additional Xiaoyuzhou transcripts now extend the knowledge base from
  agentic software engineering into harness design and automated research.
- `wiki/index.md` and `wiki/log.md` are ready for ongoing maintenance.
- Core concept and synthesis pages now show the intended style of maintenance.

### 中文
- 仓库脚手架已经搭好。
- Karpathy 的 LLM Wiki 模式已经作为第一个种子来源写入知识库。
- `AI News`、`AI Technologies` 和 `AI Equities` 已经成为模块化入口页。
- 关于 OpenClaw、vibe coding 和 AI-native 工作方式的一期小宇宙播客，已经被拆分
  进 source、concept 和 analysis 页面。
- 另外两期小宇宙转录进一步把知识库从智能体软件工程扩展到了 harness 设计和自动化
  研究。
- `wiki/index.md` 和 `wiki/log.md` 已经可以支持持续维护。
- 核心概念页和综合页已经体现出目标中的维护风格。

## Next Actions
### English
- Route new material through modules first, then split into source, concept,
  entity, and analysis pages as patterns become durable.
- Continue ingesting the backlog in `raw/inbox/`, prioritizing long-form
  sources over fragmented social posts.
- Group high-volume social captures into durable summaries instead of making one
  wiki page per low-signal fragment.
- Rebuild the index and log each maintenance pass.

### 中文
- 新材料优先先进入模块页，再在模式稳定后拆分成 source、concept、entity 和
  analysis 页面。
- 继续处理 `raw/inbox/` 里的积压内容，优先长篇资料而不是碎片化社交帖子。
- 高频社交抓取应汇总成持久摘要，而不是为每条低信号碎片单独建页。
- 每次维护后都重新生成索引并追加日志。
